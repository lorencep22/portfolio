WordPress Security Incident Recovery

Case Study: SEO Spam, User Injection & Malware Cleanup

Summary

After migrating a WordPress website (Registered Training Organisation (RTO) in Australia, specializing in vocational education and training) to a shared hosting provider, the site was compromised with SEO spam, user injection, and malicious files. I resolved the issue through a combination of manual cleanup, plugin removal, and server migration to AWS EC2.

Issues Identified

  • 167,000+ fake users registered via unrestricted registration
  • SEO metadata hijacked to promote WENGTOTO gambling site
  • Meta images replaced with adult content
  • Suspicious files found: .Admin.php, wp_settings, and altered index.php
  • Compromised plugin injected spam and possibly enabled backdoor access

Actions Taken

  • Deleted all 167k+ fake users
  • Disabled "Anyone can register" option in WordPress
  • Removed malicious files manually from the WordPress root and theme directories
  • Reverted altered SEO metadata and cleaned up spam entries in the database
  • Removed compromised plugin
  • Migrated hosting to a secure AWS EC2 instance
  • Installed Wordfence for malware scanning and firewall protection
  • Enabled 2FA, login rate limiting, and regular backups

Results

  • All SEO spam and inappropriate content removed
  • Clean, secure WordPress installation restored
  • Google indexing restored through Search Console resubmission
  • Hosting environment improved with better access control and monitoring

Technical Stack

  • Platform: WordPress 6.x
  • Server: AWS EC2 (Ubuntu)
  • Tools: Wordfence, phpMyAdmin, WP-CLI, CloudWatch
